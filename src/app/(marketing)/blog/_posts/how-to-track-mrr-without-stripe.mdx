Stripe makes MRR tracking easy. But millions of SaaS products use PayPal, Gumroad, Paddle, or LemonSqueezy — and many early-stage founders bill manually. Here's how to track MRR accurately regardless of your payment processor.

## Why MRR Tracking Is Harder Without Stripe

Stripe was built with subscriptions as a first-class feature. Other processors treat subscriptions differently:

- **PayPal** — Subscription billing exists but API data is inconsistent
- **Gumroad** — Memberships supported but no native MRR calculation
- **Paddle** — Good subscription support, but MRR dashboard is limited on lower plans
- **LemonSqueezy** — Growing subscription feature set, but newer
- **Manual billing** — Wire transfers, invoices — no automated tracking at all

## Method 1: Manual Spreadsheet

1. Create a spreadsheet: Customer, Plan, Monthly Amount, Start Date, Status
2. Each month, sum all Active rows
3. Track changes manually: new (+), upgrades (+), downgrades (-), cancellations (-)

For annual plans, divide by 12:
```
Annual plan $1,200/year → $100 MRR contribution
```

## Method 2: Processor CSV Export

- **PayPal:** Reports → Statements → Download
- **Gumroad:** Analytics → Export (memberships)
- **Paddle:** Reports → Subscriptions → Export
- **LemonSqueezy:** Reports → Subscriptions

## Method 4: Normalize Everything to Monthly

The most common mistake is double-counting annual plans.

- **Wrong:** Customer pays $1,200 upfront → record $1,200 MRR in January
- **Right:** Divide by 12 → record $100 MRR each month

## The One Number That Matters

```
Net New MRR = New MRR + Expansion MRR − Contraction MRR − Churned MRR
```

If this number is positive every month, your business is growing.
